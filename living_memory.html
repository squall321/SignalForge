SLIDE 01 · UNIFIED CONCEPT

사진을 "파일"이 아니라 "그날의 증인"으로

갤럭시는 이미 그날의 날씨 · 일정 · 음악 · 동선 · 동행자를 안다.
Living Memory는 이 흩어진 단서들을 사진과 연결하고, 사용자의 짧은 음성 한 마디까지 더해 AI가 그날의 narrative를 다시 짜내는 추억 증강 인프라다.

① Capture (촬영)
사진 + 옵션 음성
셔터 직후 3~5초 음성 윈도우가 잠깐 열림. 한 마디 남기거나, 안 남기면 자동 닫힘 — 완전 옵션
→
② Augment (증강)
컨텍스트 자동 결합
위치·시각으로 날씨·일출몰·달위상 자동 보강. 캘린더·재생음악·헬스 데이터 cross-reference.
→
③ Recall (회상)
AI narrative 재구성
사용자가 6개월·1년 뒤 사진을 다시 볼 때, LLM이 모든 단서를 종합해 그날을 자연어로 풀어냄

Context Layer — 외부 데이터 자동 보강

위치·시각만 있으면 알 수 있는 날씨·캘린더·음악·활동을 사진에 자동으로 묶는다

기대 결과물

모든 사진이 그날의 객관적 맥락(날씨·일정·음악·활동)을 자동으로 갖는다 — 사용자 부담 없이 기록 자체가 깊어진다

🌤️
⚠ 전제 조건 · 확인 필요 항목
1. 기상 데이터 접근 — 사내 기상 API 활용 또는 외부 라이선스.
2. 앱 간 권한 정책 — 캘린더·헬스·음악 접근에 대한 OS 권한 모델.
3. 저장 위치 — 외부 보강 데이터를 사진에 임베드할지, 별도 sidecar로 둘지 결정.

Voice Memo — 셔터 직후 한 마디

가장 자연스러운 인간의 추억 기록 방식인 음성으로, 그날의 기분과 단서를 잠깐 남긴다

병행 진행
완전 옵션
🎙️
셔터 누른 직후 작은 마이크 아이콘이 3초간 깜빡임 — "오늘 진짜 추웠어" 한 마디만 던지면 끝. 안 던지면 그냥 닫힘
💡
모든 사진에 강제하지 않는다. 사용자가 *"이 순간은 남기고 싶다"* 할 때만 자연스럽게 한 마디. 평소엔 존재감 없음.

기대 결과물

사용자가 부담 없이 "이 순간"의 기분을 남길 수 있는 채널 — 6개월 뒤 목소리를 다시 들으면 그날이 즉시 떠오른다

🎙️
⚠ 전제 조건 · 확인 필요 항목
1. UX 검증 — 거슬리지 않는 윈도우: 셔터 직후 음성 아이콘 노출 방식 사용자 테스트.
2. 프라이버시 — 음성 데이터는 on-device 저장이 기본. 외부 전송 시 명시적 동의.
3. Motion Photo 컨테이너 확장 — 별도 오디오 트랙 추가 호환성 검증.

기대 결과물

모든 사진이 "그날의 이야기"를 한 줄로 가짐 — 사용자가 직접 적지 않아도 AI가 단서를 모아 회상문을 만든다

✨
⚠ 전제 조건 · 확인 필요 항목
1. EXIF+ · Context Layer · Voice Memo 선행 — 입력 단서가 있어야 narrative 가능.
2. 과장 방지 — LLM이 없는 사실을 지어내지 않도록 가드레일 설계.
3. 사용자 수정 권한 — narrative는 *제안*일 뿐, 사용자가 수정·삭제 자유.
4. AI 질문 톤 — "오늘 어땠어요?" 같은 자발 질문이 부담스럽지 않은 UX 검증.

Memory Trigger — 능동적 회상 유도

저장된 메타데이터를 활용해 갤러리가 먼저 추억을 꺼내준다 — "1년 전 오늘", "비슷한 계절"

📍

▸ 입력

  • 날짜 · 위치 · 날씨 · 동행자
  • 음성 메모 · narrative
  • 현재 시각·위치·날씨 (실시간)
▸ 처리

  • 현재 컨텍스트와 과거 매칭

▸ 출력

  • 갤러리 홈 화면 카드
  • 잠금화면 위젯 (옵션)
  • 사용자가 빈도 조절 가능

기대 결과물

갤러리가 먼저 추억을 꺼내주는 능동적 인터페이스가 된다 — 사용자가 찾지 않아도 그날이 다시 떠오름

🔔
⚠ 전제 조건 · 확인 필요 항목
1. 알림 빈도 정책 — 너무 자주면 거슬리고, 너무 드물면 효과 없음. 사용자 조절 가능 UX.
2. 슬픈 추억 필터링 — 가족의 마지막 사진 등 *조심해야 할* 추억 자동 식별 필요.
3. 알고리즘 투명성 — "왜 이 추억을 꺼냈는지" 사용자가 이해 가능해야 함.

Living Memory 로드맵

다섯 세부 과제는 모두 "사진이 그날을 안다"는 하나의 비전 아래 단계적으로 작동한다.
EXIF+ 인프라를 토대로, Context Layer와 Voice Memo가 단서를 모으고, Augment Engine이 narrative를 만든 뒤, Memory Trigger와 Mood Map이 사용자가 추억을 다시 만나는 새로운 길을 연다.

# 세부 과제 역할 전제 조건 단계
A Context Layer 외부 데이터 자동 보강 기상 API · 앱 권한 정책 🟡 먼저 시작
B Voice Memo 셔터 직후 음성 한 마디 UX 검증 · 프라이버시 정책 🔵 A와 병행
C Augment Engine LLM narrative 생성 A·B 선행 · 가드레일 설계 🟣 핵심 응용
D Memory Trigger 능동적 회상 유도 A·C 선행 · 알림 정책 🔴 확장
E Mood Map 감정·계절·장소 탐색 A·B·C 선행 · 갤러리 UI 통합 🟢 새 인터페이스

🎯 Living Memory · 사진이 그날을 안다

이 과제의 본질은 "흩어져 있는 그날의 단서들을 사진 한 장에 모아, 시간이 지나도 그날을 다시 떠올릴 수 있게 한다"는 것이다.
Phase 1에서는 Context Layer(A)와 Voice Memo(B)로 단서를 모으고, Augment Engine(C)이 narrative를 만들어 사용자에게 즉시 가치를 전달한다.
이후 Memory Trigger(D)와 Mood Map(E)이 갤러리를 *수동 저장소*에서 *능동 추억 인터페이스*로 진화시킨다.

EXIF+가 사진의 물리적 자기인식을 만든다면, Living Memory는 사진의 맥락적 기억을 만든다.
