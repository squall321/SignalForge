## 1. 각 키 발급 절차

### 1-1. Slack Incoming Webhook (약 3 분)

1. https://api.slack.com/apps → **Create New App** → **From scratch**
2. App 이름 `SignalForge Alerts`, 워크스페이스 선택
3. 좌측 **Incoming Webhooks** → 토글 ON → **Add New Webhook to Workspace**
4. 채널 선택 (예 `#sf-alerts`) → **Allow** → Webhook URL 복사 (1 회만 노출)
5. URL 형식: `https://hooks.slack.com/services/T<team>/B<bot>/<secret>`

### 1-2. Groq API Key (약 2 분, 무료 권장)

1. https://console.groq.com → Google SSO 가입
2. **API Keys** → **Create API Key** → 이름 `signalforge` → 키 복사 (`gsk_...`)
3. 권장 모델: `llama-3.3-70b-versatile` — 무료 daily 14,400 req / 30 RPM 충분
4. 보안: 키는 1 회만 노출. `.env` 외부에 저장 금지. 노출 시 console 에서 즉시 revoke.

### 1-3. Bluesky App Password (약 5 분)

상세는 `BLUESKY_GUIDE.md` 참고. 요약:
1. https://bsky.app → Sign up (이메일 + 핸들 + 6 자리 코드)
2. **Settings → Privacy and security → App Passwords**
3. **Add app password** → 이름 `signalforge-collector` → 19 자 패스워드 즉시 복사
4. 일반 패스워드 사용 금지 (DM 접근 권한 노출 위험)

## 3. 적용 (서비스 재시작)

backend `Settings()` 는 시작 시 1 회 캐시 → `.env` 갱신 후 backend 재시작 필수.
crawler dispatcher / LLM provider / Bluesky 토큰은 매 호출마다 env 재읽음 → 재시작 불필요.

```bash
cd /home/koopark/claude/SignalForge
scripts/down.sh && scripts/up.sh

# 또는 backend 만 재시작:

    > /tmp/sf_backend.log 2>&1 &

bash scripts/activate-channels.sh
```


### 4-4. Backend cross-check

```bash
curl -s http://127.0.0.1:18000/api/v1/_internal/key-status | jq .summary
# 외부 Groq 까지 ping (daily limit 1 회 소모):
curl -s 'http://127.0.0.1:18000/api/v1/_internal/key-status?ping=true' | jq .
```

응답의 `summary.all_ok=true` + `groq.reachable=true` + `slack.enabled=true` 면 정상.
키 자체는 응답에 노출 안 됨 (prefix/suffix 4 자만).

## 5. 트러블 슈팅 (자주 발생)

| 증상 | 원인 | 해결 |
|---|---|---|
| `groq: fail` HTTP 401 | API Key 오타 / revoke 됨 | Groq console 에서 키 재발급 → `.env` 갱신 |
| `groq: fail` HTTP 429 | daily 14,400 / RPM 30 초과 | 분당 호출 제한, 또는 OpenRouter 로 전환 |
| `slack: fail` 형식 | URL `https://hooks.slack.com/` 시작 아님 | App 페이지의 Incoming Webhook 탭 URL 다시 복사 |
| `backend: slack mismatch` | settings 캐시 만료 안 됨 | backend 재시작 (`pkill -f uvicorn` + `nohup ...`) |
| Bluesky `createSession` 401 | 핸들에 `@` 포함 / 일반 패스워드 사용 | 핸들에서 `@` 제거, 앱 패스워드로 교체 |
| Bluesky `429` 반복 | 동시 다중 인스턴스 / `QUERY_TERMS` 과다 | `crawler/platforms/bluesky.py:MIN_DELAY` 상향 |
| Bluesky `collected=0` 키 정상 | 검색어가 사용자 모수와 불일치 | `QUERY_TERMS` 에 영어 단어 (예: "Pixel") 임시 추가 |
| 키 입력 직후 Slack 폭주 | 적체 알림 한꺼번에 발사 우려 | 5 분 dispatcher 가 분당 10 건 / 50 건 batch 로 보호 — 안전 |

## 6. 키 미입력 환경 동작 (CI / staging)

| 슬롯 | 미입력 시 동작 |
|---|---|
| `EXTERNAL_*` (3 개) | LLM 라우터 `high-shared` (local `qwen2.5:14b`) 폴백, 보고서 정상 생성 |
| `ALERT_WEBHOOK_URL` | `alert_events` INSERT 정상, dispatcher 가 `slack:dry` 라벨 부여 (POST 없음) |
| `BLUESKY_HANDLE` / `PASSWORD` | `crawl()` 빈 리스트 + 경고 로그, 파이프라인 죽지 않음 |
| 전체 미입력 | `key_health_check` → `skipped` exit 0, 시스템 모든 기능 정상 |
